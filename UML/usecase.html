Use case

Mis on Use case

Tarkvara- ja süsteemitehnikas on use case potentsiaalne stsenaarium ,
mille puhul süsteem saab välise päringu (nt kasutaja sisendi) ja vastab sellele.
Kasutusjuhtum on tegevuste või sündmuste etappide loend, mis tavaliselt määratleb
rolli (tuntud Unified Modeling Languages ​​(UML) kui tegutseja ) ja süsteemi vahel eesmärgi
saavutamiseks.

Kus kasutatakse use case

Kasutusjuhtumeid kasutatakse teadaolevalt järgmistes kontekstides:
  1. Juhtelemendina objektorienteeritud tarkvaratehnika (OOSE) ;
  2. Unified Modeling Language (UML) käitumise modelleerimise vahendina;
  3. Unified Software Development Process (UP) ja selle eelkäija, IBM Rational Unified Process (RUP) ;
  4. tarkvaranõuete spetsifikatsiooni (SRS) esialgne dokumentatsioon funktsionaalsete nõuete alternatiivse struktuurina;
  5. disaini tuletamine nõuetest, kasutades üksuse-kontrolli_piiri lähenemisviisi;
  6. ja agiilne areng .
